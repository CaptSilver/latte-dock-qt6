# Live-only QML units

Units/files that cannot be covered headlessly (and why). These are out of scope for the
headless QML gate and are the target of the live-dock capture (P3). Add entries during
curation/expansion instead of gaming a headless test.

- `plasmoid/package/contents/ui/ContextMenu.qml` — SIGSEGVs in C++ during QML finalize on construct.
- `shell/package/contents/views/WidgetExplorer.qml` — SIGSEGVs in C++ during construction.
- `plasmoid/package/contents/ui/main.qml` — `Plasmoid` attached object (PlasmoidAttached)
  can't be created without a live Applet/Containment; object can't be retained.
- `plasmoid/package/contents/ui/task/TaskItem.qml` (partial) — handlers reading `root`,
  `tasksModel`, `scrollableList`, and the `ListView` delegate `index` need a live
  containment/ListView; the pure helpers are headless-coverable.

- plasmoid/package/contents/ui/main.qml — PlasmoidItem root; its root bindings (Plasmoid.backgroundHints + the whole Plasmoid.configuration.*/formFactor/location cascade) need the C++ PlasmaQuick::PlasmoidAttached object, which Qt only creates when the item is the live root of an applet/containment. Headlessly the runtime emits 'Detected a PlasmoidItem which is not the root QML item: this is not supported', Plasmoid resolves to undefined, and createObject() returns null. No retained instance means none of its functions (getBadger, updateBadge, containsMouse, launcherExists, taskExists, previewContainsMouse, createContextMenu, mpris2 startOperation/goNext/play/pause/stop/raise/quit, sourceNameForLauncherUrl) or handlers can be called or asserted. Plasmoid is a qualified attached property (import org.kde.plasma.plasmoid), not an unqualified creation-context name, so the id:root mock pattern cannot supply it. Needs a live dock/applet (latte containment with a real Plasmoid applet backing).
